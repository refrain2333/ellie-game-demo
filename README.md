# Ellie Game Demo

为 xxhz 创作的游戏 Demo 项目集合。

## 项目愿景

将静态的图片或动图作品，逐步创作为标准的交互式游戏。

## 项目结构

```
ellie-game-demo/
├── 测试/                          # 当前测试中的游戏项目
│   ├── 迷宫游戏/
│   └── lanhu_44renxiawu41356/    # 蓝湖设计稿还原（静态页面）
│
└── 第一章/第一节/xx动画项目/       # 正式发布的游戏项目（规划中）
```

## 目录规范

- `测试/` - 开发中的游戏 Demo
- `第一章/第一节/xx动画项目/` - 正式发布的游戏项目
- 每个游戏项目包含基础三件套：
  - `img/` - 图片资源
  - `*.html` - 游戏主文件
  - `*.css` / `*.js` - 样式与脚本

## 游戏基础特征

一个标准的游戏项目通常具备：

- **交互性** - 玩家可以通过点击、拖拽、键盘等操作与游戏互动
- **规则系统** - 明确的游戏规则和胜利/失败条件
- **视觉呈现** - 画面、动画、特效等视觉元素
- **状态管理** - 记录游戏进度、分数、状态等数据
- **反馈机制** - 操作结果的可视化反馈（如路径连线、动画效果）
- **响应式适配** - 支持不同尺寸的屏幕设备

## 技术栈

- HTML / CSS / JavaScript（原生）
- SVG 图形与动画
- 响应式布局
